# Roles and Responsibilities

## Steward

**Maintains the Global Security Bootcamp Core framework.**

**Responsibilities**:
- Maintain core templates, documentation, and governance processes.
- Review and approve major changes and new program themes.
- Support organizers with best practices and guidance.
- Maintain standards compliance and consistency across events.
- Manage contributor reviews and merge decisions.

**Time commitment**: 5-10 hours/month.

## Event Organizer

**Plans and executes a local Global Security Bootcamp event.**

**Responsibilities**:
- Create and manage event repository (fork or copy core templates).
- Recruit speakers, volunteers, and sponsors.
- Customize templates for local context (agenda, communications, operations).
- Prepare labs and hands-on materials.
- Handle registration, communications, and day-of logistics.
- Collect feedback and document lessons learned.
- Contribute improvements back to core (optional).

**Time commitment**: 20-40 hours (varies by event scale).

## Reviewer

**Validates event materials against standards.**

**Responsibilities**:
- Review event agendas for alignment with program themes and standards.
- Check accessibility and inclusion considerations.
- Validate operational readiness (runbook, incident procedures, etc.).
- Provide feedback and suggest improvements.
- Approve event readiness before launch.

**Time commitment**: 5-10 hours per event.

## Readiness Checkpoints

Before your event, confirm you meet these checkpoints:

### Planning Phase
- [ ] Define event objectives and target audience.
- [ ] Confirm date, location, and organizer team.
- [ ] Identify speaker candidates and sponsor prospects.
- [ ] Create event repository with core templates.

### Preparation Phase
- [ ] Finalize agenda with sessions, speakers, and timing.
- [ ] Prepare or confirm all labs and hands-on materials.
- [ ] Create or customize slide decks and speaker notes.
- [ ] Set up registration and confirm privacy handling.
- [ ] Prepare communications plan (social, email, press release).

### Readiness Phase
- [ ] Confirm all speakers and sponsors.
- [ ] Finalize day-of runbook with contact info and procedures.
- [ ] Review Code of Conduct and reporting mechanism.
- [ ] Verify accessibility accommodations (captions, transcripts, etc.).
- [ ] Confirm tech stack and AV setup.
- [ ] Conduct readiness review against [Event Standards](event-standards.md).

### Post-Event Phase
- [ ] Collect and analyze attendee feedback.
- [ ] Thank speakers, organizers, and sponsors.
- [ ] Document lessons learned and improvements.
- [ ] Delete or anonymize personal registration data per privacy policy.
- [ ] Archive event materials in your repo or linking system.
